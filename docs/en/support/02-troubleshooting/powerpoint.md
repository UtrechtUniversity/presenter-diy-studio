# 2.3 PowerPoint problems

## The DIY Studio App reports a PowerPoint error

See errors **501** and **502** in the previous chapter.

## The PowerPoint presentation is not visible on the teleprompter

Is the teleprompter on and showing the dummy presentation?

- Return to the PowerPoint tab in the app and try opening a different presentation.
- If that works, try the first presentation again.
- If the user has only one presentation, sign out and sign in again through the **Sign in** tab.

If the dummy presentation is not visible, check the Command Prompt window for an error. Record or photograph it and restart the PC.

## The PowerPoint presentation does not appear as the video background

Check the BNC cable from the DeckLink to the Ultimatte. The left-hand DeckLink port must connect to the Ultimatte's `BACKGROUND` input. Disconnect and reconnect the cable. If no image appears, try a replacement cable.

## Other PowerPoint problems

Sign in as administrator, start PowerPoint, verify that it is activated and check:

- `Slide Show > Use Presenter View` must be selected.
- `Slide Show > Monitor` must be set to **Monitor 3 AAA**.

Both of these options must be cleared:

- `File > Options > Trust Center > Trust Center Settings… > Protected View > Enable Protected View for files originating from the Internet`
- `File > Options > Trust Center > Trust Center Settings… > Protected View > Enable Protected View for files located in potentially unsafe locations`

Restart the computer.

If PowerPoint is not activated, contact Content & Digital.
